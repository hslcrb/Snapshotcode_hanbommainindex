# 분석 결과 (Analysis Results)

## 목적 (Purpose)
본 프로젝트는 한봄고등학교 홈페이지의 특정 시점(2026.02.28)을 기능적으로 보존하고, 가독성 낮은 크롤링 데이터를 체계적인 디렉토리 구조로 재정비하는 것을 목적으로 합니다.

This project functional preservation of the Hanbom High School website at a specific point in time (2026.02.28) and reorganization of the crawl data into a systematic directory structure.

## 구조 정비 상세 (Reorganization Details)

### 1. 물리적 폴더 재구성 (Physical Restructuring)
원본 크롤러가 생성한 도메인 기반의 복잡한 폴더 구조를 인간이 읽기 쉬운 계층 구조로 개편했습니다.
The complex domain-based folder structure generated by the crawler was reorganized into a human-readable hierarchy.

- **`hanbom-h.goesw.kr`** → **`website_archive/`**
- **기능별 분류**: `pages`, `assets_common`, `assets_web`, `uploads` 등으로 에셋을 분리 배치했습니다.
  - Assets were separated into functional units like `pages`, `assets_common`, `assets_web`, and `uploads`.
- **외부 리소스 내재화**: Google 관련 폴더들을 `external_assets/` 아래로 통합 정리했습니다.
  - Google-related folders were consolidated under `external_assets/`.

### 2. 상대 경로 시스템 (Relative Path System)
원본의 절대 경로(`/...`) 체계를 모두 상대 경로로 변환했습니다.
All original absolute paths (`/...`) were converted to relative paths.

- **장점 (Benefits):** 
  - 웹 서버 없이도 개별 파일 열람 가능 (Offline accessibility without a web server).
  - 프로젝트 이동 시 경로 문제 발생 방지 (Portability across different environments).

## 기술적 관찰 (Technical Observations)
- **CMS 종속성**: 원본 사이트는 `.do` 매핑을 사용하는 Java CMS 기반이나, 본 스냅샷은 이를 정적 파일로 완벽히 대체하여 독립 동작하도록 구성되었습니다.
  - While the original site used a Java CMS via `.do` mappings, this snapshot replaces them with static files for independent operation.
- **위젯 시스템**: `widgEdit` 등 커스텀 위젯 스크립트들이 로컬 폴더로 재배치되어 메인 페이지의 동적 요소들이 정상적으로 로드됩니다.
  - Custom widget scripts like `widgEdit` were relocated to ensure dynamic elements load correctly.

## 결론 (Conclusion)
정비된 구조와 최적화된 경로 시스템을 통해, 본 아카이브는 단순한 데이터 백업을 넘어 실제 활용 가능한 디지털 자산으로 기능합니다.
With optimized paths and an organized structure, this archive serves as a functional digital asset beyond simple data backup.
